Creator walkthrough

Follow the story game flow.

01

Create a hero from a drawing or idea.

Open the Character tab. Choose whether to upload a drawing or use the prompt builder. Add the hero name, what they are, their personality, and the visual style.

  • Use clear character details like hair, eyes, outfit, ears, species and colors.
  • Pick the same visual style you want scene cards to use later.
  • Save the hero before moving to the next step.

02

Choose the title, audience, genre and game vibe.

Before the cover is made, tell OPREALM who the story is for and what kind of experience it should feel like. This helps the system build the right tone, scenes, choices and endings.

  • Keep the title short and memorable.
  • Choose Audience (Who will play it), Genre (Story type), and Game Vibe (Mood and energy).
  • Add a short Story Direction (What should happen) so the story tree has a strong starting point.

03

Generate the game cover and approve the prompt.

OPREALM combines the saved heroes, title, audience, genre, vibe and story direction into a cover prompt for approval. The generated cover should feel like the game box art for the whole story.

  • Check the generated cover prompt before spending credits.
  • Make sure there is no unwanted preset text overlay.
  • Use Enlarge and Download to inspect or save the cover.

04

Generate the story tree before making images.

The story engine creates a reusable movie-style story map with a beginning, choice branches, plot twists and different endings. This is the plan that every scene image follows.

  • The map can include happy, sad, tragic, cliff-hanger, satisfying and fulfilling outcomes.
  • Each scene gets a short Story Moment (What happens here) and choices.
  • Connected dots show how one scene leads to the next.

05

Create 16:9 scene cards one by one.

Each scene image is generated from the story tree, saved hero references, locked visual style and previous approved scene images. Approve the image, edit it with a prompt if needed, then create the next scene.

  • Keep Character Consistency (Same face, outfit, eyes, hair, ears, colors and style) locked on.
  • Use previous scene cards as continuity references so the world does not drift.
  • Use Enlarge before approving and Download if you want to save the image.

06

Add Banner UI, text and choice buttons.

Pick a UI theme, choose the banner and button pieces, then add the story text. The text box, banner and buttons can be positioned and resized so the card stays readable.

  • Use Text Size (Small, Medium, Large), Text Color and Text Alignment (Left, Center, Right).
  • Keep narration and questions short so they fit inside the banner.
  • Each button should match the choices already planned in the story tree.

07

Preview the playable story and publish safely.

Open Preview to test the full player journey. Check every choice, every ending, every image and every text layer before publishing to the reviewed OPREALM marketplace.

  • Make sure every button has a destination.
  • Check that all content is safe, kind and age-appropriate.
  • Publish only when the story feels complete and playable.

Final check

Preview, approve and publish safely.

Before publishing, play through the story, check each scene image, confirm the text is readable, and make sure every choice is kind, age-appropriate and safe. Published creations go into the OPREALM marketplace/library review flow, not open chat.
